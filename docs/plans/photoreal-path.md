# 通往照片/电影级的思路（Photoreal Path）

> 这是"怎么达成照片级"的**战略思路**文档，不是逐条任务清单（任务进度见 `photoreal-progress.md`，阶段定义见 `photoreal-overhaul.md`，验收标准见 `../specs/photoreal-acceptance.md`）。
> 锚点照片：`../refs/references.md`（L1 = 雅鲁藏布航拍：奶绿松石江 + 缠山云带 + 湿润深绿坡 + 远处雪峰 + 大气透视）。

## 一、现状诚实诊断（2026-06-20）

经过本轮迭代，画面已从"原型灰盒"清理到"干净但仍像游戏"：曝光已校准、钢轨已金属化、水已不透明、构图（FOV 78 + 关 DoF）已稳、体积云已落地。但对照 L1，仍**不像照片**，逐元素看短板：

| 元素 | 现状 | 离照片差在哪 |
|---|---|---|
| 山体 | 苍白低模折面（"泡沫塑料山"） | **核心硬伤**：30m DEM 在第一人称掠射角下读成大三角折纸 |
| 天空 | 顶部一块平灰阴天盖子 | 缺分层积云带 + 大气透视；云读成 2D 盖子而非有体积有明暗 |
| 植被 | 几乎没有（boulder proxy 已删） | 林芝是湿润密林，画面里是裸坡 |
| 江水 | 不透明但是平面、镜面感 | 缺 flow normal / depth fade / 真实急流泡沫，不流动 |
| 过山车 | 金属质感对了，但几何是 Cube 拉伸 | 车厢/钢轨是基本体，近看不是真车 |
| 成像 | 基本中性 | 缺电影级 grade / bloom / 运动模糊调校 / 大气颗粒 |

## 二、统御原则（这套思路的内核）

这四条决定"先做什么、怎么判断"，比任何单项技术都重要——它们是本会话 pivot 复盘的提炼：

1. **按"视觉贡献度 × 性价比"排序，不按难度排序。** 整图同时在多个轴上失败时，把最难的轴（地形几何）修到完美，画面仍 FAIL，且其他轴的噪声会淹没你对地形改动的判断。**先打便宜且杠杆大的（天空/大气、调色、植被遮挡），把画面拉到 70 分，再回头啃最难的。** 此前 30+ 轮卡死，正是因为反过来了。
2. **数据/资产做真相，代码别硬凑。** 真实感来自真实输入（真 DEM、扫描岩壁/植被资产、真材质），不来自把程序化噪声调得更花。能用真实资产的地方就别用 proxy 染色。
3. **每一刀对着参考照片打分，不靠"感觉好点了"。** L1 是锚。截图必须和参考并排判分（`photoreal-acceptance.md` 的 D1–D8 维度），杜绝单帧自我安慰。
4. **on-rails 是优势不是限制。** 玩家只走一条固定轨道、只看视锥走廊。**只需要把"相机会扫到的走廊"做到照片级**，远场可以靠大气透视 + 低频资产糊过去。这把"照片级整座峡谷"的不可能问题，缩小成"照片级一条带"的可解问题。

## 三、达成照片级的分层路径（按杠杆排序）

每一项给出**真正的解法**，不是"再调一下参数"。

### 杠杆 1 — 天空 + 大气透视（最便宜、最大杠杆）
参考里 ~40% 画面是云和大气。这是"照片感"的第一来源。
- **体积云做出层次**：别停在引擎 `m_SimpleVolumetricCloud` 的均匀盖子；提亮 albedo（~0.9）、降 coverage、加 scale/形状变化做出**分层积云带**，让云有迎光面/阴影面、缠在山腰不同景深。
- **大气透视（aerial perspective）**：开 `SkyAtmosphere` 的 aerial perspective + 体积雾，让远山随距离蓝灰渐隐。**这一步同时帮地形**——折面的山被空气一糊，远中景的瑕疵自然消失（呼应原则 4）。

### 杠杆 2 — 光照与调色（便宜）
- 物理太阳（已 120000 lux）+ 校准曝光（已做）+ **湿润色板**：深绿坡、不透明奶绿松石江、暖谷底；脱离"泡沫塑料白"。
- 这是 albedo + 大气的问题，不是改地形几何。

### 杠杆 3 — 植被（中等成本，但对"林芝感"决定性）
裸坡永远不像林芝。
- **引入真实植被资产**（Megascans / Fab / CC0 的针叶林、灌木、草），用 PCG/HISM 沿走廊散布，密度对标 L1 的湿润密林。
- **附带巨大好处**：密林会**自然遮住近坡的 heightfield 折面**——很多"地形不够好"其实可以被植被覆盖解决，而不是去重建地形几何。

### 杠杆 4 — 近景山墙几何（最难，留到最后，且只做走廊）
这是卡死 30+ 轮的核心。见第四节深挖。

### 杠杆 5 — 江水
- 不透明奶绿松石（已做）+ **flow map normal**（沿河道流动）+ **depth fade**（岸边变浅）+ 真实**急流白水泡沫**几何。让它流动、有深度，不是一块玻璃。

### 杠杆 6 — 过山车本体
- 车厢/钢轨/支撑现在是 Cube 拉伸 + 金属材质。**换真实 mesh**（管轨 + 箱型梁 + 车体），金属材质已对，几何换上即接近真车。这是近景常驻物体，值得做。

### 杠杆 7 — 成像收尾
- TSR + 按车速调 motion blur + 电影级 color grade（LUT）+ 克制的 bloom + 轻微大气颗粒/暗角。最后 5% 的"电影感"。

## 四、核心硬问题深挖：地形几何天花板

**为什么卡了 30+ 轮**：源数据是 30m Copernicus DEM。在**第一人称眼平、贴着近坡掠射**的角度下，30m 一个采样点的网格必然读成大三角折面——这不是 LOD/法线/材质能修的 bug，是**表示方法的固有上限**。忠实地把 30m DEM 烤成 mesh（无论 Landscape 还是 Nanite），都继承这个分辨率贫困。

**正确思路 = 分景深、分手段，别再忠实 meshing 整张 DEM：**

- **远景山体**：DEM 宏观形状是对的（真实雅鲁藏布轮廓）。配合杠杆 1 的大气透视，远山被空气糊住，折面不可见 → **DEM 够用，不动**。
- **中景坡面**：杠杆 3 的密林 + 草覆盖 → **植被遮住折面**，不需要几何级精度。
- **近景走廊岩墙（相机真正贴脸掠过的那一小条）**：这才是必须真几何的地方，也只有这一小条。解法二选一/二合一：
  1. **真实扫描岩壁资产**（Megascans cliff/rock）按视锥走廊摆放/雕刻，Nanite 承载高模，盖住 DEM 表层；
  2. **程序化 sub-30m 岩石位移**——脱离 DEM，在近崖注入 DEM 没有的层理/裂隙/崩积高频几何（Nanite displacement）。
  关键认知（外审钦定）：**"脱离 DEM"是核心指令**——不要再把 30m 网格烤得更细（已验证只会更假），而要在近景**发明** DEM 不含的细节。

一句话：**把"照片级整座 30m 峡谷"的死问题，拆成"远景靠大气、中景靠植被、近景一小条靠真实岩壁/Nanite 位移"——每一块都可解。**

## 五、验收方式（防止再次自欺）

- **锚定参考打分**：每张英雄帧与 L1 + R2/R3 并排，按 `photoreal-acceptance.md` 的 D1–D8 维度评分，不靠"感觉"。
- **英雄帧 + 沿轨道抽样**：不只盯 `WaitSeconds 12` 一帧（会诱导单帧 gaming）；沿全环抽几个机位，确保走廊整体过关。
- **离屏快循环**：`offscreen-shot.ps1` 出图 → Read → 对照打分 → 改 → 再出图。每轮记进 `photoreal-progress.md`。
- **二元硬门槛已降级**：原"全轨道无折面"绝对 gate 改为分层验收（远/中景轮廓 + 大气/植被软化即可，仅近崖要高细节）——别让几何完美主义再阻塞整体推进。

## 六、务实取舍

- **80/20**：天空+大气+调色+植被（杠杆 1–3）能把画面从"游戏"拉到"很接近照片"，成本远低于地形几何。**先把这 80 分拿到手。**
- **on-rails 红利**：永远只为视锥走廊买单。远场不值得堆几何/材质——交给大气透视和低频资产。
- **真实资产优先于程序化**：到了植被/岩壁这步，引入 Megascans/Fab/CC0 真扫描资产的性价比，远高于继续调程序化噪声。
- **过山车本体**别忘了：它是唯一近景常驻物，换真 mesh 的边际收益高。

---

## 下一刀建议（承接当前状态）
按杠杆顺序，**先做杠杆 1（天空云带 + 大气透视）补完**——它最便宜、最大杠杆，且会顺带压住远中景地形折面，让后续判断有干净信号。然后杠杆 2（调色）、杠杆 3（真实植被）。地形几何（杠杆 4）放到植被之后，且只做近景走廊一小条。
