# 画质基础（从零开始）

> 给项目负责人的入门读物。目标：让你看懂"为什么这帧假"、能当 agent 的判官。
> 每节结构：**直觉类比 → UE 里叫什么 → 你项目里对应什么 → 怎么看出不对**。
> 配套：审查时用 `AGENTS.md` / `photoreal-acceptance.md`；这里只讲原理。

## 0. 一张画面是怎么"算"出来的（全局心智模型）

把渲染想成**一台虚拟相机拍一个虚拟世界**，每一帧大致这几步：

```
几何(形状) + 材质(表面) + 灯光  →  计算每个像素收到多少光(HDR, 高动态范围)
                                  →  色调映射 tonemap (把超亮的范围压进显示器能显示的范围)
                                  →  抗锯齿 + 后期(运动模糊/景深/bloom)
                                  →  屏幕上的最终像素
```

**"照片级"= 这条链每一环都对，且喂进去的几何/材质/灯光本身够真。** 任何一环错，整帧就"假"。你项目现在的问题是：几何是方块(假)、材质是平涂(假)、灯光量级不对(假)、抗锯齿/后期建在错地基上。下面逐环拆。

---

## 1. PBR：物体表面的几个"旋钮"

**直觉**：现实里你区分"塑料/金属/木头/湿石头"，靠的是它们**怎么反光**。PBR(Physically Based Rendering，基于物理的渲染)就是用几个标准旋钮描述任何表面怎么反光，全行业统一：

- **Base Color（基础色 / albedo）**：物体"本身的颜色"，不含光影。一块红砖的红。
- **Metallic（金属度）**：是不是金属。几乎只取 0(非金属，如石头/木头/塑料)或 1(金属，如钢轨)。这个开关一变，反光方式完全不同。
- **Roughness（粗糙度）**：表面多粗糙。0 = 镜面(光滑钢/水面)，1 = 全漫反射(粉笔/土)。**这是让东西"有质感"最关键的旋钮**。湿石头粗糙度低(亮)、干土粗糙度高(哑)。
- **Normal（法线贴图）**：见 §7，用一张贴图"假装"表面有凹凸细节，而不真的加几何。

**UE 里**：材质(Material)的 Base Color / Metallic / Roughness / Specular / Normal 输入引脚。
**你项目里**：`M_CoasterTint`(`create-coaster-materials.py:196`)只给了一个纯色 Base Color + 一个常数 Roughness=0.88 + Specular=0.10 → 所以钢轨看起来像**哑光塑料**：金属度没给(=0，当成非金属)、粗糙度 0.88(很哑)。真实钢轨应该 Metallic=1、Roughness 低且有变化(磨损处不同)。
**怎么看出不对**：金属没有"方向性高光"(那种随你移动而滑动的亮条)、所有表面一样哑、像玩具 → 材质旋钮没设对。

## 2. 光的强度与曝光（像真相机一样）

**直觉**：现实的光强差异巨大——正午太阳比室内亮**几万倍**。相机靠**曝光**(光圈/快门/ISO)把这个巨大范围压成一张正常照片。渲染也一样：先用**物理量级**的光照算出真实亮度，再用**曝光**压到显示器。

- 光的物理单位：太阳这类用 **lux**，晴天正午约 **10 万 lux 量级**。
- **曝光(Exposure / Eye Adaptation)**：相机/人眼自动适应明暗。UE 里用 **EV(曝光值)** 表达。可以自动(进暗处变亮，像眼睛适应)或手动锁定。

**关键**：光照量级和曝光是**配套**的。你把太阳设成真实的 10 万、曝光也按真实标定，PBR 材质才会得到正确的明暗比例，金属/阴影/反弹才"对"。

**你项目里**：太阳 `SetIntensity(32.0)`(`CoasterRideActor.cpp:152`)——这个数远低于物理日光量级；然后又把曝光**锁死** min=max=1.0(`:121-124`)硬把画面拉亮回来。结果：**能量比例全错**——太阳和天光的相对强度不真实，所以无论怎么调，PBR 都给不出"晴天直射"的真实感。这是你画面"说不出哪里假但就是假"的底层原因之一。
**怎么看出不对**：阴影该深的地方发灰、该亮的高光不够亮、整体"平"没有立体感 → 光照量级/曝光没标定。

---

## 3. 颜色空间与色调映射（为什么会发橙/发灰）

**直觉**：渲染内部算的是**线性 HDR**(亮度可以远超 1，物理真实)，但显示器只能显示有限范围(LDR，0–1)的 **sRGB**。中间必须有一步**色调映射(Tonemapping)**把 HDR 压成好看的 LDR——就像胶片有自己的"色彩响应曲线"。UE 默认用 **ACES filmic** 这条电影感曲线(你代码里的 `FilmSlope/Toe/Shoulder` 就是在调它)。

还有个常见坑：**贴图的颜色空间**。
- Base Color(给人眼看的颜色)要标 **sRGB**。
- Normal / Roughness / Mask(是数据不是颜色)要标 **线性(关掉 sRGB)**。
标错 → 颜色或凹凸全错。(你的 `create-coaster-materials.py` 里每张贴图都显式设了 srgb 标志，就是在管这件事。)

**你项目里**：bug 日志里那个"山坡发橙"——表面看是颜色不对，根因其实是材质管线静默回退到了错材质(headless 假成功)。但**颜色发暖/发灰**这类问题，十有八九出在"光照量级 + 色调映射 + 颜色空间"这三件事的组合上。
**怎么看出不对**：整体偏色(发橙/发灰/发绿)、亮部"糊"成一片白或暗部死黑 → tonemap/曝光/色彩空间问题，不是某个贴图的事。

## 4. 全局光照 GI（为什么真实场景"通透"）

**直觉**：光不只直射，还会**到处反弹**——阳光照到绿草地，草地把绿色的光弹到旁边的白墙上，白墙微微发绿。这种"间接光/反弹光"叫**全局光照(Global Illumination, GI)**。没有 GI，阴影里是纯黑、物体像贴在背景上，一眼"游戏感"；有 GI，暗部有柔和的环境色、东西"融"进环境里，才像真的。

**UE 里**：**Lumen** = UE5 的实时 GI + 反射系统(你项目级已开，`DefaultEngine.ini` 的 `DynamicGlobalIlluminationMethod=1`)。它让阴影里有反弹光、让水面/金属有真实反射。
**你项目里**：地基(Lumen)开了是对的，但因为灯光量级不对(§2)、场景全是假几何，Lumen 没东西可正确反弹，价值发挥不出来。等阶段 A/B 修好地形和光照，Lumen 的效果才会显现。
**怎么看出不对**：阴影/背光面死黑、物体像"浮"在背景上没有环境融合 → GI 没起作用或被错误光照淹没。

## 5. 天空与大气（你"远景如画"的命根子）

**直觉**：天为什么是蓝的、远山为什么发蓝发淡？因为阳光穿过**大气**被散射。近处空气少、影响小；越远空气越厚，远山就被一层蓝雾"洗淡"——这叫**大气透视(Aerial Perspective)**，是人眼判断"远"和"大"的核心线索。峡谷之所以"如画"，一大半靠这个。

**UE 里**：
- **Sky Atmosphere**：物理模拟大气散射 → 真实蓝天 + 地平线渐变 + 大气透视。
- **Volumetric Cloud**：真正的体积云(有厚度、被阳光穿透)，不是贴图。
- **Sky Light**：天空作为一个大柔光源照亮场景(暗部的蓝调)。
这三个 + 方向光(太阳)是**一个系统**，要一起调。

**你项目里**：你有 `SkyAtmosphere`(`:157`)，但同时还手搓了**顶点色假天穹 + 扁平假云 + 朝相机的假远山**(`BuildSkyDome/BuildCloudLayer/BuildDistantRidges`)——两套打架。第一张参考图里那个"塑料天 + 方块云"就是假的那套。阶段 A/B 要删掉假的、只留物理天空 + 体积云 + 大气透视。
**怎么看出不对**：远山和近景一样清晰锐利(没有变蓝变淡)、天空是死板渐变、云像贴纸 → 缺大气透视/物理天空。

## 6. 抗锯齿与 TSR（"照片 vs 游戏"的分水岭）

**直觉**：屏幕是方格像素，斜边会出现**锯齿/阶梯**(aliasing)。而且草叶、细栏杆这种高频细节在你移动时会**闪烁爬行**。解决靠**抗锯齿**。现代实时渲染用**时序抗锯齿**：把连续好几帧的信息累积平滑——既消锯齿又消闪烁。

**UE 里**：**TSR(Temporal Super Resolution)** = UE5 的时序抗锯齿/上采样，是 5.x 默认。代价是快速运动时可能有轻微**残影(ghosting)**，需要权衡。
**你项目里**：你是**高速过山车第一人称**，运动极快——没有 TSR，植被和细线会闪成一片，立刻"游戏感/廉价感"。这是阶段 E 的成像收尾重点。
**怎么看出不对**：边缘有锯齿、远处草/栏杆在动时闪烁麻点 → 抗锯齿没开或没用 TSR。

## 7. 几何与资产（形状本身够不够真）

- **多边形(polygon)**：3D 模型由三角面组成，越多越细。一个引擎自带 Cube 只有 12 个三角面——所以你的"轨道/车"是方块。真实钢轨要几百上千面的扫掠剖面。
- **Nanite**：UE5 的"虚拟化几何"。以前高模太贵不能进游戏，要手工做低模 + LOD(远处换低模)。Nanite 让你**直接用电影级超高模**，引擎自动按需显示。这是 UE5 能做照片级的支柱之一。你项目级已开(`Nanite.ProjectEnabled=1`)，但没喂给它高模资产。
- **贴图(Texture)**：包在模型表面的图片。一套 PBR 资产通常有 Base Color/Normal/Roughness/AO 几张配套贴图。
- **法线贴图(Normal Map)**：一张特殊的紫蓝色贴图，用**颜色编码表面朝向**，让平面"看起来"凹凸不平(砖缝、岩石纹理)而不真加几何。**§1 提的旋钮之一，提升质感性价比最高。** 你的地形材质现在**没接法线**(因为之前 moire bug 断开了)→ 山坡软塌塌没细节。
- **资产来源**：照片级要真扫描资产。免费且高质的：**Quixel Megascans(随 UE 免费，Fab 上)**——真实扫描的岩石/植被/地表。这是把"几何和材质够真"一次性解决的捷径。

**怎么看出不对**：物体边缘是直棱角(多边形太少)、表面平滑无细节(没法线贴图)、近看糊(贴图分辨率低或平铺) → 几何/贴图/资产不够。

## 8. 地形：Landscape 与高度图

**直觉**：游戏里的山地不是一个个模型，是一张**高度图(heightmap)**——一张灰度图，**越白越高**，引擎据此把一张网格"顶"成地形。
**UE 里**：**Landscape** 系统。高度图分辨率越高、来源越真，山形越可信。
**你项目里**：Yarlung 当前已经不是 UE `ALandscape` 渲染路径；可见地形是由 DEM/r16 源数据生成的 `SM_YarlungCorridorTerrain` StaticMesh。DEM 负责中远景真实山体轮廓，近中景照片级细节靠 Nanite 岩壁、森林资产和分层 PBR 材质接管。
**怎么看出不对**：山形规律/对称/泛泛、不像真实地质；或近看地表有平铺重复、网格/等高线纹路 → 高度图太假/太糙，材质没分层。

## 9. 把它串起来：照片级 = 全部对齐

回到 §0 那条链，照片级需要**同时**满足：
1. **几何真**(Nanite 高模 + 真实 DEM 地形) ← §7 §8
2. **材质真**(PBR 旋钮设对 + 法线 + 分层) ← §1 §7
3. **光照真**(物理量级 + 曝光标定 + GI) ← §2 §4
4. **天空/大气真**(物理天空 + 体积云 + 大气透视) ← §5
5. **成像真**(色调映射 + TSR + 后期相机感) ← §3 §6
6. **资产够好 + 有人用眼力盯着** ← §7 + 你

缺任何一条，画面就在某处"露馅"。你项目现在 1–5 全部欠债，所以是原型；好消息是**地基(Lumen/Nanite/VSM)都开了**，欠的是上层。计划 `photoreal-overhaul.md` 的阶段 A→F 就是按这个顺序逐条补。

## 10. 最少必要的学习路径

你不用全会。当好判官，**精读 §1–§6**(质感、光、颜色、GI、天空、抗锯齿)就够看穿八成"假"。然后:
- 看 **William Faucher**(YouTube，UE 灯光)的灯光+后期几期。
- 下一个 **Epic 官方 photoreal 示例场景**，进去看它怎么打光、材质怎么分层、后期怎么设——照着抄一遍，眼力涨最快。
- 每学一节，回项目对应阶段验证一次(学完 §2 就去把太阳量级改对、曝光放开，看截图变化)。
