# CoasterSim Docs

Live docs are intentionally small. For normal Yarlung AAA work, read only:

1. `../AGENTS.md` - operating rules and commands.
2. `plans/photoreal-progress.md` - current status, latest evidence, next task.
3. `plans/photoreal-plan.md` - active product, visual, and architecture plan.
4. `specs/photoreal-acceptance.md` - scoring and done definition.
5. `refs/README.md` and `refs/local/` - reference images.

Archive policy:

- `archive/` and `reviews/archive/` are archaeology. Do not read them by default.
- Open archived files only when a current doc points to a specific file or when
  investigating an old decision.
- Do not add long transcripts to live docs. Keep live progress concise and move
  stale detail into `archive/`.
