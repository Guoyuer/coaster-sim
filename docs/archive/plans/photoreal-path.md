## 2026-06-20 路线裁决：必须换轨

当前失败不是“天空不够好”或“台阶还差一点”，而是**世界空间模型错了**：

- 过山车第一人称画面必须像在真实雅鲁藏布峡谷中飞行，能读到江水、对岸山体、峡谷纵深、云带和植被。
- 旧路线把主要山体压成一条轨道扫出来的 corridor strip / DEM-derived 近景墙。截图里表现为蓝色空洞、低模折纸墙、相机被近坡吞掉。
- “沿同一份 DEM/r16 source 继续细分/调 LOD/调法线”已被多轮验证为低天花板；它只能让同一个错误模型更细。
- 未来轨道还会有起伏、airtime、甚至大回环。只按 2D 轨道偏移生成一张带状地形，不可能覆盖所有相机姿态。

**结论：技术路线从“调同一份 DEM-derived terrain / corridor strip”切到“连续 UE 峡谷世界底座 + 视锥走廊资产化造景”。**

---

## 新核心策略

1. **连续峡谷世界先成立。**  
   UE 场景里必须先有连续、不自交、不会吞相机的峡谷基础几何。可以裁到轨道视野所需范围，但不能用一条会折叠的轨道带状 mesh 代替世界。

2. **轨道路线服务第一人称构图。**  
   轨道生成不能只满足净空和 G 值；还要满足视锥质量：前方不能长期被近坡堵死，英雄段要看见江谷纵深、对岸崖壁、远山和天空。必要时轨道应更高架、更临江，而不是贴地钻坡。

3. **DEM 只负责宏观轮廓。**  
   30m DEM 可以提供山谷大形和远景高度关系，但不能承担近景照片级岩壁。近中景靠真实资产/作者化几何接管。

4. **资产优先，程序化只做辅助。**  
   近景岩壁、植被、轨道/支撑、水体必须逐步换成真实/扫描/作者化资产。程序化噪声只能补 breakup，不能替代 cliff/forest/water 的主体形态。

5. **每步以真实第一人称截图判定。**  
   `offscreen-shot.ps1` 的 1440p 截图必须 Read 打开看；日志 PASS 不能当视觉 PASS。

## 分层实现路线

## L2/L3 第一人称参考驱动路线图

两张第一人称参考图给出的目标不是“峡谷旁边有一条轨道”，而是一个稳定的电影构图：

- 车厢/扶手占底部 20–30%，轨道从画面中心进入右侧山体，给速度和尺度。
- 轨道贴着一侧陡坡或森林岩壁飞行，但前方和左侧必须打开，能持续看到江谷纵深。
- 江水在画面中下方或左侧成为连续引导线，不是偶尔露出的一块水面。
- 对岸山体必须有山脊、沟谷、森林斑块、湿岩面和远近层次；不能是单一灰绿坡。
- 远景需要雪峰/高山剪影、云带和大气透视，形成“雅鲁藏布/林芝”而不是普通低模山谷。

因此后续路线按“第一人称镜头分镜”设计，而不是按地图铺满：

1. **Station / 出站预备段**：低速看见车体、扶手、前方第一道临崖弯；视野中必须预告江谷和远山。
2. **Outbound 高崖俯冲段（英雄段）**：轨道沿右侧森林岩壁下切，左前方打开江谷；这一段是第一优先级，必须先做到像参考图。
3. **River-view S 弯**：轨道轻微左右摆动，让江水、对岸山脊、近侧林坡交替进入画面；避免长时间正对空白坡。
4. **Turnaround 大弯**：在最深峡谷处做可读的大半径转向，同帧看到来路轨道、江湾、对岸山体。
5. **Return 高台段**：从更高一侧回看江谷和远雪山，减少近坡遮挡，给全程视觉喘息。
6. **Brake / 回站段**：收束到森林边和岩壁边，保留真实地形尺度，不突然回到灰盒场景。

技术落地顺序也按这个分镜排：

1. **先做英雄段视锥盒**：只为 Outbound 12s 左右这段建立高质量 canyon set piece，范围约相机前方 300–800m、左右各 200–600m、远景到江谷/雪峰方向。走廊外不花预算。
2. **山体主体从“单张 DEM 坡”换成“组合资产地貌”**：近侧用作者化/Nanite 湿岩壁和森林坡，中景用连续山脊/沟谷 mesh，远景继续用 DEM base。
3. **植被先解决中远景读法**：需要坡面 canopy mass 和林带轮廓，而不是把小 shrub proxy 撒密。没有真树/森林资产时，不把 proxy scatter 当作通过路径。
4. **轨道成为真实前景资产**：车头/扶手/轨道截面/护栏必须优先真实化，因为参考图里它们是照片可信度的第一层。
5. **再扩展全程**：英雄段过关后，把同一套“临崖轨道 + 江谷开口 + 森林岩壁 + 远山云带”模板复制到 S 弯、Turnaround、Return 高台段，每段都用第一人称截图验收。

禁止项：

- 不再用巨大 rock_face proxy 贴片糊脸；已验证会读成低模幕墙。
- 不再把 shrub proxy 放大/加密冒充森林；它在截图里只变成绿噪点。
- 不再为整张方形地图平均铺资产；只服务第一人称可见路线。

### 1. 连续峡谷底座（当前第一优先级）

- 用 DEM-led Nanite static mesh 建立连续 canyon base。
- 保留宏观山形和河谷位置；避免 corridor strip 自交、近轨大洞、相机被地形吞掉。
- 生成成本可接受：当前 1009x1009 base mesh 约 1.0M verts / 2.0M tris，commandlet 约 31s，适合快速迭代。

出口标准：多时间点第一人称截图不再出现蓝色空洞、轨道钻入近坡、带状边界、条带自交。

### 2. 轨道视锥路线重建

- 轨道生成器新增/使用“视觉约束”：临江、可见对岸、可见远山、可见天空，前方不能被近坡长期堵死。
- 保持 `verify-track-clearance.py` 硬门禁：净空、lat/vert/long G、速度、airtime、坡度必须 PASS。
- 当前试验方向：沿 thalweg/江谷中心线，高架净空约 300m 级，先换取真实峡谷视野，再回头优化支撑和近景资产。

出口标准：英雄段和抽样段能读成沿江峡谷高架过山车，而不是贴山矿洞。

### 3. 近中景照片向组合形态

参考 L2/L3，视锥内山体要从“单一 DEM-derived 坡面”变成组合形态：

- 缓坡 / talus apron：低频坡面、崩积碎石、植被过渡。
- 山脊 / buttress：凸出的岩脊和层状岩体，不是单一平墙。
- ravine / gully：顺坡冲沟、湿暗水线、局部雾气。
- 湿岩崖面：冷灰/灰绿、高粗糙度、局部水迹和阴影。

实现上优先用 Nanite cliff/rock/forest assets 或作者化 mesh patches；不要继续把 30m DEM 烤得更细。

### 4. 水体、植被、轨道资产化

- 江水：乳白青绿、流动 normal、白水/泡沫、岸边 depth fade。
- 植被：林芝湿润深绿密林，PCG/HISM/foliage 沿视锥走廊铺开。
- 轨道/车/支撑：从 cube/巨柱换成真实钢轨剖面、桥墩/桁架/车体资产。

这些是照片感的主体，不应再由 `ACoasterRideActor` 直接生成永久美术。

### 5. 天空/光照/成像收尾

天空、大气透视、调色、运动模糊仍重要，但它们不能掩盖错误几何。顺序是：

1. 空间和路线成立；
2. cliff/forest/water/track 主体成立；
3. 再用天空、大气、ACES、TSR、motion blur 把画面收成电影感。
