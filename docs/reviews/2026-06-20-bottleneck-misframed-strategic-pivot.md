## 一、我看到了什么（参考 vs 现状）

**真实参考 L1**，按画面视觉占比：
- ~40% 是**戏剧性的云带 + 大气透视**（云缠在深色山腰、远处雪峰若隐若现）——这是"照片感"最大的单一线索。
- 一条**不透明的奶绿松石色**江水，带白色急流/泡沫，嵌在切出来的河床里。
- **湿润的深绿森林坡 + 暖色农田 + 桃花**，饱和、有层次、有暖冷对比。

**当前渲染**（`cleanup-river-width-v1`、`c2-wide-void-v1`）：
- 天空是**一片空的蓝色渐变**，零云、零大气透视。
- 山是**苍白发灰的低模折纸**，读成泡沫塑料/雪堆。
- 水是**一块半透明玻璃板**，带等高线纹路，像镜子不像河。
- 第一人称正中是一排**细立柱 + 下垂电缆**，像篱笆/电线幕，占满中景。
- 整体是**去饱和的白/冰蓝单色调**——读成"冬天/雪"，不是"湿润的喜马拉雅春季峡谷"。

> 注：`*-void-*` / `diag-*` 是诊断隔离图，空天是故意的；但**非诊断的** `cleanup-*` 同样是空天 + 白折山 + 立柱幕 + 玻璃水，所以这些结论与诊断隔离无关，是真实现状。

## 二、真正的诊断：这张图同时在 5 个独立维度上失败

按"对'不像照片'这个判决的贡献度"重排优先级（**不是**按文档当前的关注度）：

| 排序 | 真正的失分项 | 文档当前的处理 | 修复成本 |
|---|---|---|---|
| **1** | 没有云 / 大气透视（参考里占 ~40% 画面） | **写进了禁区**"不要再调 SkyAtmosphere"；B 阶段挂着"仍缺云带"未做 | 中，杠杆最大 |
| **2** | 调色 / 反照率全错（泡沫白 vs 湿绿松石） | 没系统处理，被当成"换地形的隐性回归" | **便宜** |
| **3** | 立柱+电缆篱笆占满第一人称中景 | 调过间距 72→144m，仍毁画面 | **便宜**（该先问：POV 里要不要露这么多支撑） |
| **4** | 水是透明玻璃板，不是奶绿急流 | 反复迭代材质，仍是 translucent ribbon | 中 |
| **5** | 地形折面 / 台阶 | **烧掉 30+ 轮 + 设硬门槛 + 设禁区** | **最贵、最难** |

**关键洞察：就算把第 5 项（台阶）修到完美，这张图依然 FAIL——因为 1~4 全在拖。** 这正是卡死的机制：当整张图在 5 个轴上同时崩，把最难的轴优化到极致，得到的仍是 FAIL 的图——于是**永远拿不到正反馈，也永远分辨不出地形改动到底有没有用**（其他四项噪声太大，信号被淹没）。这是对着最贵的变量做局部优化。

## 三、为什么当前流程注定卡死（三个自缚）

1. **二元硬门槛 +"全轨道任何地方不许有台阶/折面"** —— 给**最难的变量**加了一个**绝对约束**，会无限期阻塞其他一切进展。照片级是连续、整体的判断，不该被一个几何完美主义 gate 劫持。
2. **禁区把天空/大气封死** —— 文档自己承认"缺云带/大气透视"，这恰恰是参考里最大的视觉线索，却禁止去碰。等于主动放弃杠杆最大的那一刀。（禁区①的本意是"别用调色掩盖台阶"，但它被泛化成了"不许做天空"，误伤。）
3. **只盯一张英雄帧 + 掠射低角构图** —— 参考是**航拍俯瞰**，英雄帧是**轨道上的平视低角**。任何 heightfield 在眼平掠射角下都会读成烂面。可能在追一个**这个构图永远追不上的目标**。最美的过山车 POV 往往是**仰望耸立崖墙**或**越过山谷望向被大气包裹的远景**，不是贴着近坡擦过去。

## 四、建议的下一步（按顺序，便宜+高杠杆优先）

**先停止迭代地形。** 做一轮"establishing shot"整体过关，把便宜+高杠杆的轴拉到位，让画面先到 70 分，**然后**再用一张"其余都已照片级"的帧去裁决台阶——那时才有干净信号。

1. **解封天空：加体积云 + 大气透视。** 对照 L1。`UVolumetricCloudComponent` + 调 `SkyAtmosphere`（散射/Rayleigh/Mie + 地平雾），让远山有"隔着空气"的蓝灰渐隐。这是单刀杠杆最大的一项；做完画面"照片感"会比过去 30 轮加起来都明显。**这一步要求先松动禁区①**（见第五节）。
2. **整体脱离"泡沫塑料白"调色/打光。** 湿润深绿坡、不透明奶绿松石江、暖色谷底。检查 `M_YarlungMeshTerrain` 的 `TerrainBrightness`/反照率（现在被刷成白青）、`M_YarlungCorridorTerrain` 顶点色、太阳色温/SkyLight 强度。这是 albedo+光照问题，便宜。
3. **干掉第一人称里的立柱篱笆。** `ACoasterRideActor` 的支撑生成：要么大幅减密度/改成稀疏纪念碑式桥墩并移出视锥中心，要么调整相机使其不糊脸。真实过山车 POV 不会有一道电线幕挡在正前方。
4. **江水改不透明奶绿 + 白色急流泡沫，嵌进河床。** `M_YarlungRiverWater`：降 translucency、加 depth fade + flow normal，别再是漂在地面的玻璃板。
5. **最后**再裁决地形折面。大概率会发现：① 它没想象的致命；② 配合大气透视 + 植被遮挡，远中景折面被自然压住；③ 真正要根治的只是**英雄段近崖一小块**，用真实岩壁资产/Nanite displacement 局部解决，**不必重烤整张地形 mesh**。

## 五、对硬门槛与禁区的修正建议

- **松动"全轨道第一人称无台阶/无折面"硬门槛 → 改为分层验收**：远/中景只要求"轮廓正确 + 被大气透视和植被自然软化"；**只有英雄段近崖**保留高细节要求。绝对几何 gate 不该凌驾于整体照片感之上。
- **禁区①拆分**：保留"不许用调色/几何遮盖去**掩盖**台阶"的本意；但**明确允许并优先**为照片级本身去做天空/大气/体积云/整体打光。"做对的天空"和"用调色掩盖台阶"是两回事，现在的禁区把前者一起误杀了。
- 其余禁区（不做 procedural overlay 伪替换、不把规则 heightfield 细分当根治）保留，依然成立。

## 通读项目 · 结构性"不合理"清扫 — 2026-06-20（已实施 + 截图验证）

用户指令：通读项目，把挡着照片/电影级的不合理设定通通改掉。逐文件读 `CoasterRideActor.cpp`（相机/后处理/光照）、`DefaultEngine.ini`、`create-coaster-materials.py`、`YarlungRiverActor.cpp`、`YarlungSceneryActor.cpp` 后，定位并修复 6 处硬伤（全部 build + offscreen 验证，截图 `pivot-sweep-v1`/`pivot-sweep-hero-v2` 已 Read，无回退）：

1. **过山车 = 哑光塑料**：`ApplyVisualMaterials()` 用 `M_CoasterTint`（Roughness 0.88、Metallic 0）把钢轨/支撑/枕木渲成非金属哑光灰。→ 给 `M_CoasterTint` 加 `Metallic` 参数（默认 0，岩石不受影响），钢轨设 Metallic 1.0/Rough 0.24 抛光钢、支撑 0.9/0.45 喷漆钢、枕木 0.85/0.55 镀锌、车厢深红。**这是过山车本体从不像真车的头号原因。**
2. **景深 DoF 对焦 42m 把山景模糊掉**：`DepthOfFieldFocalDistance=4200cm` + 开启 → 与史诗远景相反。改为关闭 DoF，全画面锐利。
3. **FOV 92° 过广畸变**：放大近景杆子、偏游戏感。→ 78°，更电影、近景畸变小。
4. **江水 Opacity 0.14 = 玻璃板**（86% 透明）。→ Opacity 0.92 近不透明、奶绿松石色、Rough 0.34 乳浊不镜面。
5. **曝光偏热 + scenery 岩石反照率过高**：诊断证明白不在地形顶点色（已是暗绿）。`RockOutcrops/CliffFaces` 用 `M_CoasterTint` 平涂、albedo 0.23~0.28 ≈ 地形 3 倍 → 过曝爆白。→ 岩石 albedo 降到 ~0.10 湿岩范围；曝光 bias +0.95→-0.25。
6. **SkyLight=3.0 环境光过抬、阴影发平**。→ 1.1，恢复电影级阴影对比。

**诊断工具留存**：`CoasterRideActor.cpp` BeginPlay 新增命令行旋钮 `-YarlungExposureBias=` / `-YarlungSkyLightIntensity=` / `-YarlungSunIntensity=`，可不重 build 快速 A/B 定位光照问题（本次用它定位了"白来自曝光/材质而非反照率"）。

**裁定**：6 项都是真·结构性不合理、修复无回退、画面更干净（金属轨道/不透明水/稳构图/平衡曝光）。但**这是"便于照片级"的清障，不是达成照片级**——剩余主短板仍是：山体几何/材质（低模折面 + 偏白）、支撑立柱密度（cut #3 未做）、水仍是平面无 flow、云偏平。下一步按 pivot 顺序继续 cut #3（砍支撑）→ 山体真实资产/几何。
